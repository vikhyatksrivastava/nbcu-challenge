Peacock Data Engineering Technical Evaluation

There are three parts to this evaluation
0. (Optional) Data preparation. This type of work is common on our team. If you are not able to complete this part, that's okay! A processed data set is provided to work with in the subsequent parts.
1. Review the data and questions provided.
2. Build a visualization that provides insights into the data.

Background

Suppose we have a library where we require patrons rate the titles they read on a scale from 1 to 5. These ratings are collected in a series of JSON messages. The aim of this exercise to understand the titles and the viewership patterns.

Part Zero

The JSON messages are received at the end of every day. The format of each message is

{
  'id': 'User ID',
  'item': 'Title Rated',
  'rating': 'User Rating',
  'datetime': 'Date of Rating'
}

The file ratings.tar.gz contains a JSON object with messages from the past 6 months. The aim of this part is to organize this data into a format usable for visualization and analysis.

Part One

You can either use the data generated in Part Zero or you can utilize ratings_clean.csv.gz. Here are some questions to get things started.
1. What are the most popular titles?
2. What groups of patrons are similar?
These are broad questions that you should expand upon. What other insights can you share?

Part Two

In the visualization tool of choice, build a collection of visualizations to provide insights to the questions above and other items you uncover. Once this is complete, package all code, documentation, and visualization into a zip file.
